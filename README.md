# Пет-Проект: Система Управления Задачами (To-Do List) для Сотрудников

## Описание

> Этот пет-проект представляет собой базу данных для учета задач (To-Do List) и проектов у сотрудников компании. Цель проекта — продемонстрировать навыки проектирования реляционной базы данных, включая определение сущностей, атрибутов и связей между ними.

## Применение в бизнесе
Данная база данных подходит для реализации как в IT - компаниях, так и в других сферах деятельности. Благодаря более структрированному подходу к учету рабочих проектов/задач данная система помогает оптимизировать время выполнения проектных заданий, не терять задачи сотрудникам, приоритизировать задачи, понимать статус проекта и сотрудника.

## Функциональность

Данная БД реализует следующие задачи:
*   Учет информации о сотрудниках.
*   Создание, хранение и отслеживание задач и проектов.
*   Назначение задач сотрудникам.
*   Связывание задач с проектами.
*   Управление статусами и приоритетами задач и проектов.

## Структура Базы Данных

(Опционально, но рекомендуется) Вставьте сюда **ER-диаграмму** (если создадите), или **опишите таблицы** и их связи. Пример:


### Сущности и Атрибуты:
*   **Employee:** `employee_id`, `firtsname`, `lastsname`, `middle_name`, `date_of_birth`, `email`, `work_phone`, `department`, `position`, `status`
*   **Tasks:** `task_id`, `title`, `description`, `due_date`, `task_status`, `priority_id`, `created_by`, `assigned_to`, `revue_by`, `project_id`, `comments`
*   **Projects:** `project_id`, `title`, `description`, `start_date`, `finish_date`, `manager_of_project`, `prepared_by`, `project_status`, `title`
*   (и так далее для всех сущностей)

### Связи:
*   Сотрудник (Employee) *создает* много Задач (Task) (1 ко многим).
*   Задача (Task) *принадлежит* одному Проекту (Project) (Много к одному).
*   Задача (Task) *имеет* много Тегов (Tag), и Тег (Tag) *имеет* много Задач (Task) (Много ко многим, через вспомогательную таблицу `Task_Tags`).

## Использование / Установка

(Опционально) Если вы добавите SQL-скрипты для создания и наполнения базы данных, укажите, как их использовать:
1.  Клонируйте репозиторий: `git clone <URL_вашего_репозитория>`
2.  Установите СУБД (например, PostgreSQL, SQLite).
3.  Выполните скрипт `schema.sql` для создания структуры БД: `psql -U username -d database_name -f schema.sql`
4.  (Опционально) Выполните скрипт `data.sql` для добавления тестовых данных.
